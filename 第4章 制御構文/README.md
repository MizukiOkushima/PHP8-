### 練習4.2
#### 1.while命令とdo〜while命令との違いを説明してください。
while命令は条件式を前置判定するのに対して、do〜while命令は後置判定します。<br>
その性質上、条件式が最小からfalseである場合、while命令はループを一度も処理しませんが、do〜while命令はいかなる場合も一度はループ処理を実行します。

### 練習4.3
#### 1.現在の周回をスキップする命令、現在のループを脱出する命令を、それぞれ答えてください。
スキップ：continue命令<br>
脱出：break命令<br>
continue、breakはいずれもfor、while、do〜whileなどの繰り返し構文の中で使用できる命令です。<br>
その性質上、continue、break命令はif命令などの条件分岐と組み合わせて利用するのが一般的です。<br>
#### 2.リスト4.32のコードをwhile命令で書き換えてみましょう。
```PHP
<?php
$sum = 0;
for ($i = 1; $i <= 100; $i++) {
  if ($i % 2 !== 0) { continue; }
  $sum += $i;
}
print "合計値は{$sum}です。";
```
```PHP
<?php
$sum = 0;
$i = 0;
while ($i <=100) {
  $i++;
  if ($i % 2 !== 0) { continue; }
  $sum += $i;
}
print "合計値は{$sum}です。";  // 結果：合計値は2550です。
?>
```
while命令では、カウンター変数$iをインクリメントするタイミングを謝ると無限ループになる可能性があります。<br>

### この章の理解度チェック
#### 1.for命令とcontinue命令とを使って、1〜100の範囲にあるき数値の合計を求めてみましょう。
```PHP
<?php
$sum = 0;
for ($i = 1; $i <= 100; i++) {
  // 2で割り切れる場合（偶数）、continueでスキップ
  if ($i % 2 === 0) {
    continue;
  }
  $sum += $i;
}
print "合計値は{$sum}です。";
?>
```

#### 2.リスト4.31をwhile命令を使って書き換えてみましょう。
```PHP
<?php
$sum = 0;
for ($i = 1; $i <= 100; $i++) {
  $sum += $i;
  if ($sum > 1000) { break; }
}
print "合計が1000を超えるのは、1～{$i}を加算したときです。";
```
```PHP
<?php
$sum = 0;
$i = 0;
while ($i <= 100) {
  $i++;
  $sum += $i;
  if ($sum > 1000) {
    break;
  }
}
print "合計が1000を超えるのは、1～{$i}を加算したときです。";
?>
```
カウンター変数を利用したループは原則としてfor命令で表現すべきですが、ここでは練習のためにwhile命令を利用しています。<br>
while命令では、カウンター変数の加算／減算のタイミングに気をつけてください。<br>
特にbreak、continue命令を利用している場合、記述場所によってはカウンター変数が変化せず、無限ループになる恐れがあります。<br>

#### 3.次のコードはforeach命令を使って配列の値を一律に1.5倍するためのコードです。空欄①〜⑤を埋めてコードを完成させてください。
```PHP
<?php
$data=  ① 10,25,50 ② ;
③ ($data as ④ ) { $item ⑤ 1.5; }
print_r($data);  //結果：Array([0]=>15[1]=>37.5[2]=>75)
```
①[<br>
②]<br>
③foreach<br>
④&$item<br>
⑤*=<br>
ポイントは④です。foreachループの中で配列を直接操作するには仮変数を参照渡しする必要があります。<br>

#### 4.変数$languageの値が「Python」「PHP」「Ruby」であれば「インタプリター言語」「C#」「Java」であれば「コンパイル言語」と表示するスクリプトを、switch命令を使って作成してください。
```PHP
<?php
$language = 'Python';
switch ($language){
  case'Python':
  case'PHP':
  case'Ruby':
    print'インタプリター言語';
    break;
  case'C#':
  case'Java':
    print'コンパイル言語';
    break;
}
```
複数のcase句をbreakせずに実行させるフォールスルーは原則として禁止ですが、このように複数の条件式を表すために空のcase句を連ねる方法はよく使われます。覚えておくとよいでしょう。<br>

#### 5.4のスクリプトをif命令を使って書き換えてみましょう。
```PHP
<?php
$language = 'Python';
if ($language === 'Python' || $language === 'PHP' || $language === 'Ruby'){
  print 'インタプリター言語';
} elseif ($language === 'C#' || $language === 'Java'){
  print 'コンパイル言語';
}
```
||演算子ではなく、elseifブロックを条件値の数だけ記述してもかまいませんが、コードが冗長になるだけなので、通常は避けるべきです。<br>
また、ここでは練習のためにif命令を利用していますが、このようなケースではswitch命令を優先して利用してください。<br>

#### 6.4のコードをmatch式を使って書き換えてみましょう。
```PHP
<?php
$language = 'Python';
print match($language){
  'Python','PHP','Ruby'=>'インタプリター言語',
  'C#','Java'=>'コンパイル言語'
};
```
match式はPHP8で導入された新しい構文です。<br>
しかし、配下の式が1つであれば、switchに比べても、同じことをシンプルに表現できます。<br>
それが許される環境であれば、積極的な利用をお勧めします。<br>

#### ColumnPHPの歴史──永久欠番のPHP6と、最新のPHP8
PHP5の後、当初はPHP6のリリースが予定されており、実行エンジンの内部処理がUTF16ですべて置き換えられる予定でした。しかし、予想以上に大規模な変更になったことから開発は滞り、さまざまな問題も生じたことから、2010年に結局、開発は断念されることになりました。その後、PHP6で予定していた機能の多くは、PHP5のマイナーバージョンに取り込まれ、PHP6は永久欠番となったのです。<br>
その後、実に10年以上ぶりのメジャーバージョンアップとなったのは、2015年12月にリリースされたPHP7です。PHP7のポイントは、なんといっても型宣言（6.1.5項）の本格的なサポート。引数／戻り値などに型を明示的に宣言できるようになった点です。型宣言によって、クラス／関数の仕様を明確に表しやすくなり、より大規模な開発にも対応しやすくなっています（もちろん、望むならば型宣言を利用しなければ良いだけなので、手軽さが損なわれるわけではありません）。<br>
そして、2020年11月にリリースされたのが、本書のターゲットである最新バージョンPHP8です。PHP8では、待望の属性（11.1節）がようやく搭載されました。本格的な活用には、PHP8そのものの普及と周辺ライブラリのサポートを待たなければなりませんが、今後、フレームワーク（第9節のColumn）を利用した開発が大きく変化していく可能性をはらんでいます。<br>
一方、バージョンを8まで重ねてきただけあって、核となる機能には下位互換性を損なうような変化はさほどありません。PHP7までで安全なコードを記述してきているならば、移行にはほぼ問題ないはずなので、積極的にPHP8の導入を検討することをお勧めします。<br>
